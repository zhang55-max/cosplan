# CosPlan - 产品需求文档 (PRD)

> 版本：V1.0 | 日期：2026-05-12 | 作者：产品团队

---

## 1. 产品定位

**CosPlan** 是一款专为 COSER（角色扮演者）打造的创作助手 APP。以「角色项目」为核心，串联 **筹备 → 拍摄 → 选片 → 修图 → 发布** 全流程，解决信息碎片化、进度失控、发布焦虑三大痛点。

## 2. 目标用户

- **主要用户**：独立 COSER，月均出 1-3 套正片
- **次要用户**：COS 社团成员，需多人协作筹备
- **用户特征**：手机重度使用、视觉敏感、社交平台活跃

## 3. 痛点分析

| 痛点 | 严重度 | 现状 | 解法 |
|------|--------|------|------|
| 筹备信息碎片化 | ⭐⭐⭐⭐⭐ | 参考图在相册、清单在备忘录、链接在淘宝 | 角色项目聚合 |
| 选片耗时 | ⭐⭐⭐⭐⭐ | 一次拍摄 200-1000 张，纯手动筛选 | 三态标记系统 |
| 与后期沟通难 | ⭐⭐⭐⭐ | 微信截图+打字，易遗漏 | 照片级修图备注 |
| 文案撰写焦虑 | ⭐⭐⭐⭐ | 每次重新构思，标签搜索费时 | 台词库+标签库 |
| 多项目进度混乱 | ⭐⭐⭐ | 同时跑 2-4 个角色，忘记缺啥 | 仪表盘全局看板 |
| 服道漏带忘买 | ⭐⭐⭐⭐ | 拍摄现场发现缺道具 | 清单勾选系统 |

## 4. 功能需求

### 4.1 首页仪表盘 (P0)
- 全局统计卡片：进行中项目数、待修照片数、未完成筹备项数、已完成项目数
- 快捷操作入口：新建角色、查看待修图
- 项目卡片列表：封面图 + 作品来源 + 进度条 + 状态标签
- 状态流转：筹备中 → 拍摄中 → 选片中 → 修图中 → 已完成

### 4.2 角色项目管理 (P0)
- 新建角色：角色名（必填）、作品来源、人设描述
- 头像上传：本地 Base64 压缩存储
- 参考图集：支持多张上传，附带备注说明
- 角色卡片列表：网格布局，显示封面+名称+来源+统计
- 角色详情：汇总该角色的筹备/照片/文案数据
- 编辑 / 删除角色（删除需二次确认）

### 4.3 拍摄筹备清单 (P0)
- 四大分类：服装、道具、场地、分镜
- 每项字段：标题、购买链接(选填)、参考图(选填)、备注(选填)
- 完成勾选：checkbox 切换已备齐/未备齐
- 展开/折叠：每分类独立控制，显示 "已完成/总计"
- 按角色筛选：顶部角色选择器

### 4.4 照片素材管理 (P0)
- 上传照片：FileReader → Canvas 压缩(最大宽800px) → Base64 存储
- 缩略图生成：200×200 裁剪
- 三种状态：原片待选(红) / 待修图(黄) / 已修完成(绿)
- 网格视图：3 列，每格显示缩略图+状态角标
- 状态筛选栏：全部 / 原片 / 待修 / 已修
- 照片详情弹窗：大图 + 状态切换按钮 + 修图备注文本框
- 统计栏：各类别照片数量
- 删除照片：详情弹窗内删除按钮

### 4.5 文案生成器 (P1)
- 台词库：按角色预设经典台词，支持增删
- 标签库：按平台(小红书/微博/B站/抖音)分类，支持自定义标签
- 文案预览：实时拼接 台词 + 作品来源 + 选中标签
- 一键复制：navigator.clipboard.writeText
- 一键重新生成：随机打乱标签顺序

## 5. 非功能需求

- 离线可用：全部数据存 localStorage，无网络依赖
- 移动端优化：触控区域 ≥ 44px，字号 ≥ 14px
- 图片压缩：单张 ≤ 100KB，总计不超 localStorage 5MB 限制
- 启动速度：首屏渲染 < 1s
- 数据安全：提供 JSON 导出/导入备份功能

## 6. 验收标准

- [ ] 可新建/编辑/删除角色，上传头像和多张参考图
- [ ] 筹备清单可按角色添加服装/道具/场地/分镜，勾选完成
- [ ] 可上传照片，切换原片/待修/已修状态，附加修图备注
- [ ] 可预设台词和标签，一键生成文案并复制
- [ ] 首页展示真实全局统计数据
- [ ] 数据刷新不丢失（localStorage 持久化）
- [ ] 手机触控流畅，按钮足够大
- [ ] 零依赖，单文件，浏览器直接打开可用
