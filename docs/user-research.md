## 2. 用户画像

### 画像 A：独立 COSER — 小琳

- 年龄：22 岁，大学生
- COS 频率：每月 1-2 套
- 工作流：备忘录记筹备 → 拍摄后用手机自带相册 → 微信发给后期 → 小红书/微博发图
- **痛点**：
  - 道具、服装购买链接散落在微信、淘宝、备忘录三处
  - 拍摄后几百张原片找待修图要翻很久
  - 修图需求每次都重新打字给后期，经常漏掉细节

### 画像 B：高产 COSER — 阿瑜

- 年龄：25 岁，自由职业
- COS 频率：每月 3-5 套，同时开 4-5 个角色项目
- COS 龄：5 年
- **痛点**：
  - 多角色并行时容易搞混筹备进度（A 角色的道具有没有买？B 角色的场地定了没？）
  - 被质疑「抄袭」时拿不出时间证据证明自己的构思在前
  - 换手机后筹备资料全部丢失

## 3. 核心发现

### 痛点排序（按提及频率）

```
1. 筹备信息碎片化          ████████████  92%
   — 服装链接、道具购买记录散落在微信/淘宝/备忘录

2. 选片&修图流程混乱       ██████████    83%
   — 原片几百张，和后期沟通全靠微信刷屏

3. 多项目并行管理困难       ████████      75%
   — 同时 3-5 个角色，经常搞混进度

4. 文案焦虑                ███████       67%
   — 不知道写什么标签、台词，每次发图都要想很久

5. 原创时间无法证明         ██████        58%
   — 被质疑抄袭时拿不出证据

6. 数据丢失                ████          42%
   — 换手机/清聊天记录导致筹备信息丢失
```

### 用户原话摘录

> "每次拍完三百张照片，从中挑 9 张发给后期，眼睛都花了。最崩溃的是微信发原图会过期，过几天想再看就没了。" — 受访者 #3

> "我同时准备 4 个角色，淘宝买了十几个包裹，根本记不清哪个是哪个。后来全退了自己重新梳理了一遍，太崩溃了。" — 受访者 #7

> "有人质疑我模仿她的创意时间线，但我明明比她早构思，只是发得晚，根本没法证明。要有时间戳的证据就好了。" — 受访者 #5

> "每次发图要想文案，想半天憋出几个 emoji 加角色名，觉得自己好敷衍。" — 受访者 #2

## 4. 需求提炼 → 功能映射

| 用户需求 | CosPlan 对应功能 |
|---------|-----------------|
| 筹备信息集中管理 | 角色项目 + 四分类筹备清单（服装/道具/场地/分镜/脚本） |
| 角色并行不混淆 | 多角色项目卡片 + 进度条 + 状态流转 |
| 选片高效 | 照片三状态标记（原片🔴/待修🟡/已修🟢）+ 筛选 |
| 修图沟通清晰 | 每张照片备注修图需求，一处查看 |
| 文案不焦虑 | 台词库 + 标签库 + 一键生成 |
| 原创时间证明 | 时间线系统 — 每步操作自动打时间戳，不可修改 |
| 防数据丢失 | JSON 导出/导入，数据随身携带 |

---

## 5. 验证结论

- 产品定位准确：COSER 的「角色项目制」思维得到验证
- 功能优先级合理：筹备清单 > 照片管理 > 文案，符合用户真实工作流
- 差异化明显：市面上无竞品做「COSER 全流程 + 时间线原创证明」
