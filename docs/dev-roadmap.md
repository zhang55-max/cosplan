# CosPlan - 开发阶段执行步骤

> 版本：V1.0 | 日期：2026-05-12

---

## 开发原则

1. **增量交付**：每 Phase 独立可运行，完成即验收，通过再进入下一 Phase
2. **单文件开发**：所有代码在 `cosplan.html` 中，按 Phase 逐步增加功能
3. **每日日志**：每天结束前更新 `dev-logs/YYYY-MM-DD.md`
4. **零依赖**：只用 HTML+CSS+JS，不引入任何第三方库

---

## Phase 1：项目骨架 + 首页仪表盘

**目标**：搭好框架，5 个 Tab 能切换，首页有基本展示

| 任务 | 细节 |
|------|------|
| HTML 骨架 | viewport meta、PWA manifest link、字符集 |
| CSS 基础 | CSS 变量、全局 reset、暗色主题、字体、间距 |
| 底部导航 | 5 Tab 按钮，点击切换 active，selected 样式 |
| 视图容器 | 5 个 `<main>` 块，display:none/block 切换 |
| JS 工具函数 | `genId()`, `loadData()`, `saveData()`, `showToast()` |
| 首页 HTML | 统计卡片(4格)、快捷操作区(2按钮)、项目列表区 |
| 首页渲染 | 从 localStorage 读数据，渲染统计数和项目卡片 |
| 空状态 | 无角色时显示引导提示 |
| 状态样式 | 进度条、状态标签的颜色映射 |

**验收检查**：
- [ ] 5 Tab 可点击切换，选中态正确
- [ ] 首页看到统计卡片（初始全为 0)
- [ ] 首页看到空状态提示
- [ ] Toast 函数可正常弹出

## Phase 2：角色项目管理

**目标**：完整的角色 CRUD + 头像/参考图上传

| 任务 | 细节 |
|------|------|
| 角色列表页 HTML | 2 列网格布局，角色卡片 |
| 新建角色弹窗 | 表单：角色名、作品来源、人设描述、头像上传 |
| 图片上传 | FileReader → Canvas 压缩(800px) → Base64 |
| 头像渲染 | 默认占位图 / 已上传头像 |
| 参考图上传 | 多张，弹窗内缩略图预览，可删除 |
| 保存角色 | 校验名称必填 → genId → push → saveData |
| 角色卡片 | 显示头像、名称、来源、参考图数、创建时间 |
| 编辑角色 | 复用新建弹窗，预填数据，保存覆盖 |
| 删除角色 | 长按或按钮 → 二次确认 → 级联删除关联数据 |
| 空状态 | 无角色时显示引导 |

**验收检查**：
- [ ] 新建角色，上传头像，保存成功
- [ ] 角色卡片正常显示
- [ ] 编辑角色，数据正确回填
- [ ] 删除角色，确认后消失，关联数据一并清理
- [ ] 刷新页面数据不丢失

## Phase 3：角色详情 + 筹备清单

**目标**：查看角色详情，管理四类筹备清单

| 任务 | 细节 |
|------|------|
| 角色详情页 | 点击角色卡片 → 展开详情视图 |
| 详情内容 | 头像大图、名称、来源、人设、参考图横滚、关联统计 |
| 筹备清单页 HTML | 角色选择器(select) + 四栏可折叠面板 |
| 添加清单项弹窗 | 表单：分类选择、标题、链接、备注、参考图(选填) |
| 清单项渲染 | 每项：checkbox + 标题 + 链接图标 + 备注 |
| 折叠展开 | 点击分类标题切换，旋转箭头动画 |
| 完成勾选 | 点击 checkbox → 切换 completed → saveData → 刷新计数 |
| 删除清单项 | 滑动或长按删除 |
| 计数显示 | 每个分类标题显示 "已完成/总计" |
| 角色选择 | 切换角色 → 重新加载该角色的清单 |

**验收检查**：
- [ ] 角色详情页正确显示所有信息
- [ ] 可添加服装/道具/场地/分镜项
- [ ] 折叠展开正常，计数正确
- [ ] 勾选完成后样式变化，计数更新
- [ ] 切换角色，清单正确切换

## Phase 4：照片素材管理

**目标**：上传照片、标记状态、管理修图流程

| 任务 | 细节 |
|------|------|
| 照片页 HTML | 角色 Tab 栏 + 状态筛选栏 + 3 列网格 + 统计栏 |
| 上传按钮 | 底部固定按钮，触发 file input |
| 照片压缩 | FileReader → Canvas(800px) → 原图 Base64 |
| 缩略图生成 | Canvas(200×200, cover裁剪) → 缩略图 Base64 |
| 存储 | Photo 对象 → push 到 cosplan_photos |
| 网格渲染 | 3 列，缩略图 cover，右下角状态圆点 |
| 状态筛选 | 全部/原片/待修/已修 四个按钮，filter 后重新渲染 |
| 照片详情弹窗 | 大图查看，状态切换按钮组，备注 textarea |
| 状态切换 | 点击切换 raw/to_edit/edited → saveData |
| 备注保存 | textarea onChange → saveData |
| 删除照片 | 弹窗内删除按钮 → 二次确认 → remove |
| 统计栏 | 实时计算各类别数量 |
| 空状态 | 无照片时引导上传 |

**验收检查**：
- [ ] 上传照片，缩略图出现在网格中
- [ ] 状态筛选正常工作
- [ ] 点击照片打开详情弹窗，大图清晰
- [ ] 状态切换和备注保存生效
- [ ] 删除照片正常
- [ ] 统计栏数字正确

## Phase 5：文案生成器

**目标**：台词库 + 标签库 + 一键生成文案

| 任务 | 细节 |
|------|------|
| 文案页 HTML | 角色选择器 + 台词列表 + 标签区 + 预览区 + 操作按钮 |
| 预设数据 | 首次使用时写入默认台词和标签 |
| 台词列表渲染 | 可点击选中(高亮)，最多选 1 条 |
| 添加台词 | 弹窗：输入台词文本 → 保存 |
| 删除台词 | 长按删除 |
| 标签库渲染 | 按平台分组，checkbox 多选 |
| 添加标签 | 弹窗：输入标签 + 选择平台 → 保存 |
| 文案预览 | 实时拼接：选中台词 + 换行 + 角色作品来源 + COS正片 + 换行 + 选中标签 |
| 一键生成 | 按模板拼接文案，显示在预览区 |
| 一键复制 | navigator.clipboard.writeText → Toast 提示"已复制" |
| 重新生成 | 随机打乱标签顺序，重新拼接 |
| 空状态 | 无台词/标签时引导添加 |

**验收检查**：
- [ ] 可添加和选择台词
- [ ] 可添加和勾选标签
- [ ] 文案预览实时更新
- [ ] 一键复制成功
- [ ] 切换角色，数据正确隔离

## Phase 6：全局串联 + 打磨

**目标**：所有模块数据打通，体验打磨，交付可用产品

| 任务 | 细节 |
|------|------|
| 首页真实统计 | 遍历所有角色计算：进行中/待修照片/未完成筹备/已完成 |
| 首页列表真实进度 | 按角色实际筹备完成率 + 照片修图率 计算进度 |
| 角色详情统计 | 点击角色详情显示真实的筹备/照片/文案数量 |
| 数据导出 | 按钮 → 收集 4 个 Key → JSON Blob → 下载 |
| 数据导入 | 按钮 → file input → JSON.parse → 合并/覆盖 → saveData |
| Toast 完善 | 所有操作(保存/删除/复制/错误)给出 Toast 反馈 |
| 删除确认 | 所有删除操作弹窗二次确认 |
| 空状态完善 | 每个页面空状态都有引导 Emoji + 提示 + 按钮 |
| 动效打磨 | 页面切换动画、弹窗动画、按钮反馈 |
| 安全区适配 | bottom-nav 适配 safe-area-inset-bottom |
| PWA | manifest.json inline, 简短 service worker 注册 |
| 最终测试 | 按验收标准逐条测试 |

**验收检查**：
- [ ] 全流程走通：建角色 → 加筹备 → 上传照片 → 标记状态 → 生成文案
- [ ] 首页统计数正确
- [ ] 数据导出再导入，完整无丢失
- [ ] 所有删除有二次确认
- [ ] 手机浏览器打开，触控流畅
